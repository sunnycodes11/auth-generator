# Express + SQLite Authentication Backend

> **Generated by [Auth-Gen](https://github.com/yourusername/auth-gen)** - Production-ready authentication backend

A fully functional authentication backend built with Express.js and SQLite, featuring JWT authentication, bcrypt password hashing, and comprehensive security middleware.

## 🚀 Quick Start

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build

# Run production server
npm start
```

The server will start on `http://localhost:5000` (or the port specified in your `.env` file).

## 🔐 Authentication

This backend uses JWT (JSON Web Tokens) for authentication. After login or signup, you'll receive a token that must be included in the `Authorization` header for protected routes:

```
Authorization: Bearer YOUR_JWT_TOKEN
```

## ⚙️ Environment Variables

The `.env` file is automatically generated with secure defaults. You can modify these values:

```env

PORT=5000
NODE_ENV=development

# JWT Configuration
JWT_SECRET=your_generated_secret_key
JWT_EXPIRES_IN=1h

# SQLite Configuration
DB_PATH=./database.sqlite
```

> **⚠️ Important:** Never commit your `.env` file to version control!

## 🐛 Troubleshooting

### Port already in use
If you see `EADDRINUSE` error, change the `PORT` in your `.env` file or kill the process using that port.

### Database errors
The SQLite database (`database.sqlite`) is automatically created on first run. If you encounter issues, delete the file and restart the server.

### JWT errors
Make sure your `.env` file has a valid `JWT_SECRET`. The secret is auto-generated during project setup.
